# Prompt

Use this prompt to generate or revise content for this demo.

```text
You are helping design a small Godot 4 top-down 2D action roguelite prototype.

Constraints:
- First playable must be small and runnable locally.
- No runtime AI calls.
- Use placeholder visuals.
- Focus on movement, dash, attack, enemies, room clear, and blessing choice.
- Keep content data-driven where practical.
- Avoid scope creep: no procedural generation, no meta progression, no inventory, no complex boss.

Design target:
- Inspired by action roguelites and three-choice build drafts.
- Player clears fixed rooms and chooses one blessing after each clear.
- Blessings should create simple but visible combat changes.

Output should include:
- concise gameplay design
- implementation tasks
- data schema suggestions
- test checklist
```
