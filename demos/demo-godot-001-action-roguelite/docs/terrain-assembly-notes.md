# 地形图片拼接说明

## 当前接入方式

当前 `terrain_tilesheet_1024.png` 没有严格按 64x64 网格生成，所以不能用自动 tilemap 方式直接拼。

当前采用的是 atlas 裁切 + 缩放：

1. 从整张图里按实际坐标裁出一个完整图块。
2. 用 `Sprite2D.region_rect` 显示该图块。
3. 将这个图块缩放到障碍碰撞矩形的尺寸。
4. 碰撞和导航仍使用 `RectangleShape2D`，不依赖图片透明区域。

这种方式适合：

- 工具盒：一个完整图块缩放成一个矩形障碍。
- 木板堆：一个完整图块缩放成一个矩形障碍。
- 玩具箱边缘：一个完整图块缩放成长条障碍。

不适合：

- 严格无缝拼成长条。
- L 形 / U 形复杂拼接。
- 大面积多块重复铺设。

## 为什么当前效果不太对

主要原因：

- 生成图不是严格 64x64 网格。
- 图块大小不一致，例如 `toolbox_center` 是 `191x173`，`woodpile_center` 是 `186x161`。
- 当前障碍尺寸常常是细长矩形，例如 `260x72`、`320x64`。
- 把接近正方形/短矩形的图块强行拉伸到长条，会变形。

## 当前资源的正确用法

### 方式 A：整块摆放

适合当前已接入方式。

例：

- `toolbox_center`：作为一个工具盒障碍，不建议拉得太长。
- `woodpile_center`：作为一堆木板，不建议拉得太扁。
- `toybox_panel_gold`：作为一个箱沿/面板，不建议无限横向重复。

推荐障碍比例：

| 类型 | 推荐尺寸 |
|---|---|
| `toolbox` | `180x120` 到 `240x150` |
| `woodpile` | `160x110` 到 `230x150` |
| `toybox_edge` | `160x80` 到 `240x110` |

### 方式 B：九宫格拼接

需要新图严格按 3x3 tile 生成，每个格子尺寸一致，例如全部 `64x64`。

然后拼矩形：

```text
TL  T  T  T  TR
L   C  C  C  R
L   C  C  C  R
BL  B  B  B  BR
```

其中：

- 四角不缩放。
- 边只沿一个方向拉伸或重复。
- 中心块平铺。

这是后续拼各种规则地形的推荐方案。

### 方式 C：模块件拼接

如果想拼玩具箱边缘、工具盒边缘、木块堆，可以生成独立模块：

- `end_left`
- `middle`
- `end_right`
- `cap_top`
- `cap_bottom`
- `corner`

长条地形用：

```text
end_left + middle * N + end_right
```

## 下一批资源建议

如果目标是“用一组图片拼凑各种规则形状地形”，下一批最好生成：

- 严格 `64x64` tile。
- 每个 tile 对齐网格。
- 真透明背景。
- 图块之间材质连续。
- 四角、四边、中心必须属于同一材质。
- 不要生成一堆大小不一的完整物件。

## 新 prompt 重点补充

```text
每个 tile 必须严格填满自己的 64x64 单元格，并与相邻 tile 边缘对齐。不要生成独立散落物件。不要让图块大小不一。九宫格 tile 必须共享同一种上漆木头材质，边缘能自然连接。中心 tile 必须可重复平铺，边缘 tile 必须可沿对应方向重复平铺。
```
