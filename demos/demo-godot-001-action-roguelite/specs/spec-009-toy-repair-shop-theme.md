# Spec: 玩具修理屋主题方向

## 背景

当前 Godot Demo 已完成：

- 3 房间动作 Roguelite 闭环。
- 赐福三选一。
- Debug 赐福面板。
- 开始 / 胜利 / 失败 UI 流程。
- 基础 M3 构筑可见反馈。

但当前主题仍偏通用动作 Roguelite，缺少清晰世界观和差异化表达。

用户决定优先沿“童话治愈 × 玩具修理屋”路径推进。

## 核心概念

一句话：

```text
夜晚的修理屋里，坏掉的玩具王国醒了。你是小小修理师，进入玩具箱王国，把发怒、断线、生锈、遗忘的玩具们修好。
```

核心基调：

- 童话
- 治愈
- 夜晚
- 温暖灯光
- 玩具箱冒险
- 修复而非杀戮
- 小小英雄对抗失控玩具

## 世界观

白天，修理屋只是街角一家安静小店。

夜晚，旧柜子、抽屉、玩具箱、积木城堡都会醒来。被遗忘、坏掉或没有修好的玩具，会在梦一样的夜里失控。

玩家不是去“杀死”玩具，而是：

- 重新上发条
- 缝好破口
- 找回纽扣心
- 擦掉锈迹
- 安抚失控情绪
- 把它们送回温暖的玩具箱

每一局 Roguelite 是一次“夜间修理”。

```text
入夜 -> 进入玩具箱王国 -> 修复失控玩具 -> 获得修理灵感 -> 继续深入 -> 黎明前完成修理
```

死亡/失败解释：

```text
小修理师的灯熄灭，被玩具箱送回修理台。下一晚重新进入。
```

## 主角定位

推荐主角：小小修理师 / 发条修理精灵。

视觉关键词：

- 小围裙
- 大号缝衣针 / 小木锤 / 发条扳手
- 背着工具包
- 头顶小灯
- 体型小，但动作灵活

当前 Demo 中玩家绿色小三角后续可替换为：

```text
小修理师占位图形：带朝向的小工具箭头 / 小木偶剪影。
```

核心动作命名：

| 当前动作 | 玩具修理屋命名 |
|---|---|
| Attack | 修理打击 / 敲正 |
| Dash | 翻滚闪避 / 发条冲刺 |
| Blessing | 修理灵感 / 工具灵感 |
| Kill | 修复完成 |
| Enemy | 失控玩具 |
| Room | 玩具箱房间 / 修理区域 |
| Victory | 黎明前修好全部玩具 |
| Defeat | 灯熄灭，被送回修理台 |

## 敌人方向

当前敌人可主题化为：

| 当前敌人 | 主题替换 |
|---|---|
| 近战红色敌人 | 断线木偶 / 生气积木兵 |
| 远程蓝色敌人 | 漏气弹簧炮 / 坏掉铁皮兵 |
| 精英近战 | 大号发条熊 / 旧木偶队长 |
| Projectile | 弹出的螺丝 / 纽扣弹 |

敌人不是死亡，而是被修好：

- 红色闪烁：失控状态被敲松。
- 燃烧可替换为“发热 / 过载”。
- 敌人消失：修复完成，回到玩具箱。

## 构筑路线

先设计 5 条主题自然长出的路线，不急着全部实现。

### 1. 发条路线：攻速 / 连击 / 过载

主题：重新上紧发条后，动作越来越快，但也可能过载。

玩法方向：

- 攻击速度提升。
- 连续命中叠加发条层数。
- Dash 后下一击加速。
- 过载时短时间爆发，随后冷却。

对应当前能力：

| 当前赐福 | 可映射 |
|---|---|
| 迅捷出手 | 发条上紧 |
| 疾风步 | 弹簧冲刺 |

### 2. 缝补路线：回血 / 护盾 / 容错

主题：缝补布偶、补回棉花，提供生存能力。

玩法方向：

- 修复敌人后回复生命。
- 受伤时获得棉花护盾。
- 低血时触发一次“补丁”。
- 清房后恢复更多。

对应当前能力：

| 当前赐福 | 可映射 |
|---|---|
| 血暖余烬 | 纽扣心 |

### 3. 胶水路线：控制 / 黏住 / 区域限制

主题：用胶水、贴纸、胶带限制失控玩具。

玩法方向：

- 攻击减速敌人。
- Dash 留下胶水痕迹。
- 敌人被控制后更容易修复。
- 区域内敌人移动变慢。

当前 Demo 暂无直接能力，可作为 M4 新增路线。

### 4. 积木路线：范围 / 击退 / 搭建

主题：积木、木板、小锤子，强调空间控制。

玩法方向：

- 攻击范围变长。
- 击退更强。
- 生成临时积木障碍。
- 大范围横扫。

对应当前能力：

| 当前赐福 | 可映射 |
|---|---|
| 长柄之势 | 长木尺 |
| 锋刃打磨 | 磨亮小木锤 |

### 5. 灯光路线：净化 / 点亮 / 追踪

主题：修理台的小灯、夜间温暖光线。

玩法方向：

- 攻击附带光点。
- 被点亮敌人受到持续修复。
- 清房后点亮区域。
- 光点自动寻找失控玩具。

对应当前能力：

| 当前赐福 | 可映射 |
|---|---|
| 余火印记 | 暖灯余光 |

## 当前 6 个赐福主题映射

| 当前赐福 | 新名称建议 | 路线 | 新描述建议 |
|---|---|---|---|
| 锋刃打磨 | 磨亮小木锤 | 积木 / 直接修复 | 修理打击力度 +20%。 |

| 疾风步 | 弹簧冲刺 | 发条 / 爆发 | 发条冲刺后，下一次修理打击力度 +50%。 |
| 血暖余烬 | 纽扣心 | 缝补 / 生存 | 修复一个失控玩具后回复 8 点生命。 |
| 余火印记 | 暖灯余光 | 灯光 / 持续修复 | 修理打击会附加持续净化。 |
| 长柄之势 | 长木尺 | 积木 / 范围 | 修理打击范围 +20%。 |

## UI 文案方向

| 当前文案 | 新文案方向 |
|---|---|
| 赐福 | 修理灵感 |
| 选择一个赐福 | 选择一个修理灵感 |
| 调试：添加赐福 | 调试：添加修理灵感 |
| 本局赐福 | 本晚灵感 |
| 胜利 | 黎明前修好了所有玩具 |
| 失败 | 小灯熄灭了 |
| 敌人 | 失控玩具 |
| 房间 | 区域 / 玩具箱 |

## 美术方向

短期仍用几何占位，但要逐步改语义：

- 玩家：绿色小箭头 -> 小修理师 / 工具剪影。
- 近战敌人：红色三角 -> 断线木偶 / 积木兵。
- 远程敌人：蓝色形状 -> 弹簧炮玩具。
- 投射物：小圆点 -> 螺丝 / 纽扣。
- 攻击矩形：黄色块 -> 木尺挥击 / 工具扫过区域。
- Damage Flash：红闪保留，但可改成“灯光晃动”效果。

## 当前阶段建议

不要立刻做复杂新系统。先做“主题替换层”：

1. 改赐福名称、描述、标签。
2. 改 UI 文案。
3. 改战斗术语：敌人 -> 失控玩具，击杀 -> 修复。
4. 在 `design.md` 新增玩具修理屋主题章节。
5. 后续再选 2 条路线做机制深化：
   - 发条路线
   - 缝补路线

## 验收标准

- [ ] 设计文档中明确玩具修理屋主题。
- [ ] 6 个赐福改为修理灵感风格。
- [ ] UI 中主要“赐福”文案替换为“修理灵感”。
- [ ] 胜利/失败文案符合童话修理主题。
- [ ] 当前 Demo 仍可完整通关。
- [ ] Debug 面板仍可添加所有灵感。
- [ ] Godot 运行无脚本错误。

## 风险

| 风险 | 影响 | 应对 |
|---|---|---|
| 只改名不改机制，主题感不足 | 玩家仍觉得是换皮 | 后续用发条/缝补做机制深化 |
| 童话治愈削弱打击爽感 | 动作不够爽 | “修理打击”仍保留敲击、弹开、过载反馈 |
| 文案过幼稚 | 受众变窄 | 保持温暖但不过分低龄 |

## 下一步

先做主题替换层：

```text
赐福 -> 修理灵感
敌人 -> 失控玩具
击杀 -> 修复完成
胜利/失败 -> 黎明/小灯熄灭
```

完成后再讨论：

- 发条路线是否作为第一条正式构筑路线。
- 缝补路线是否作为第二条正式构筑路线。
