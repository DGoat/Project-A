# Spec: 接触伤害分离与远程敌人攻击欲望调整

## 背景

M1 手测中发现：

1. 近战敌人接触并命中玩家后，会持续贴住玩家。
2. 玩家移动也难以和敌人分开，容易形成“黏住”体验。
3. 远程敌人攻击欲望偏强，房间 2 同时存在近战压力和远程弹体时，新手压力过高。

这会阻塞 M1 的最小可玩闭环验证，因为玩家还没进入“读招与走位”的体验，就先被接触伤害和弹体压死。

## 目标

- 近战敌人命中玩家后自动后退一小段，避免持续黏住。
- 玩家受伤后获得短暂无敌时间，避免连续扣血。
- 接触伤害变成有节奏的伤害，而不是贴身瞬间连死。
- 降低远程敌人攻击欲望，让房间 2 压力更可控。
- 保持实现轻量，不做完整击退/硬直系统。

## 非目标

- 不做完整受击动画。
- 不做复杂 knockback 物理系统。
- 不做敌人状态机重构。
- 不做远程敌人复杂战术 AI。
- 不做最终数值平衡。

## 范围

本次改动涉及：

1. 玩家受伤无敌时间。
2. 近战敌人命中后分离。
3. 远程敌人射击频率和距离倾向调整。
4. 问题追踪与开发日志更新。

## 设计方案

### 1. 玩家受伤无敌时间

在 `player.gd` 中增加：

```gdscript
var invulnerable_time := 0.0
var hurt_invulnerable_duration := 0.65
```

规则：

- 玩家受伤后进入短暂无敌。
- 无敌期间再次受到接触伤害或弹体伤害时忽略。
- 受伤时给玩家短暂闪色反馈。

### 2. 近战敌人命中后分离

在 `melee_enemy.gd` 中增加：

```gdscript
var recoil_time := 0.0
var recoil_direction := Vector2.ZERO
@export var recoil_speed := 260.0
@export var recoil_duration := 0.18
```

规则：

- 近战敌人接触玩家并造成伤害后，进入短暂 recoil。
- recoil 期间敌人向远离玩家方向移动。
- recoil 结束后恢复追击。

效果：

```text
敌人命中玩家 -> 敌人后退 -> 玩家获得脱身空间
```

### 3. 接触伤害节奏

由于玩家有无敌时间，接触伤害不会连续秒扣。

本次不额外给近战敌人加复杂攻击冷却。先通过：

- 玩家无敌时间
- 敌人命中后后退

共同解决“黏住连死”。

### 4. 降低远程敌人攻击欲望

调整 `ranged_enemy.gd` 默认参数：

```gdscript
@export var shoot_cooldown := 1.8
@export var preferred_distance := 300.0
```

同时可以增加首次射击延迟：

```gdscript
shoot_time = shoot_cooldown * 0.7
```

效果：

- 远程敌人不再一出现就高频射击。
- 玩家有更多处理近战敌人的时间。
- 房间 2 难度降低。

## 验收标准

- [ ] 近战敌人命中玩家后会短暂后退。
- [ ] 玩家被近战敌人贴住后，有机会移动或 dash 脱身。
- [ ] 玩家不会在 1 秒内被接触伤害连续扣多次血。
- [ ] 远程敌人射击频率明显降低。
- [ ] 房间 2 压力降低，但仍有远程威胁。
- [ ] 不影响玩家攻击和敌人死亡逻辑。
- [ ] 不影响清房、祝福选择和房间推进。
- [ ] Godot 运行无脚本错误。

## 风险

| 风险 | 影响 | 应对 |
|---|---|---|
| 敌人后退距离过大 | 敌人显得逃跑，不像近战 | 控制 recoil 时间和速度 |
| 无敌时间过长 | 游戏太简单 | 初始设为 0.65s，后续调参 |
| 远程射击太慢 | 远程敌人失去威胁 | 先降到 1.8s，后续根据手测调 |
| recoil 与追击切换抖动 | 手感怪 | recoil 期间跳过追击逻辑 |

## 实施步骤

1. 修改 `player.gd`：
   - 增加无敌时间变量。
   - `_physics_process()` 中递减计时。
   - `take_damage()` 中判断无敌。
   - 受伤后短暂闪色。
2. 修改 `melee_enemy.gd`：
   - 增加 recoil 变量。
   - recoil 期间向远离玩家方向移动。
   - 接触玩家造成伤害后触发 recoil。
3. 修改 `ranged_enemy.gd`：
   - 降低射击欲望。
   - 增加首次射击延迟。
4. 更新 `issue-tracker.md`。
5. 更新 `dev-log.md`。
6. 如确认踩坑，更新 `pitfalls.md`。
7. Godot 无头加载验证。
8. 用户手测。

## 回滚方案

如果 recoil 或无敌逻辑导致新问题：

1. 保留远程敌人调参。
2. 回滚 recoil 逻辑。
3. 只保留玩家受伤无敌。
4. 继续记录为 M2 手感问题。
