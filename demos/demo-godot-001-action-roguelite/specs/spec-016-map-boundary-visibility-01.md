# Spec 016：地图边界与角色可见性修正 0.1

## 背景

Todo P0 仍有“地图边界贴边/半露问题”。当前玩家位置只用 `global_position.clamp(play_area_min, play_area_max)` 限制，范围是整张背景图边界，未考虑角色碰撞半径与相机可视边缘，因此玩家到边界时可能半露或贴边过近。

## 本轮目标

1. 玩家不能贴到背景最外边缘。
2. 玩家在地图边界附近保持完整可见。
3. 不改动相机跟随逻辑。
4. 不影响障碍碰撞、胶水坑、Dash。

## 非目标

- 不新增地图边框美术。
- 不改 NavigationRegion2D。
- 不调整敌人出生点。

## 实现方案

### Player

将可移动区域从整图边界收缩为带 margin 的区域：

```text
play_area_min = Vector2(80, 120)
play_area_max = Vector2(3088, 1264)
```

理由：

- 横向留出约 80px，避免角色贴边半露。
- 顶部留出约 120px，避免 HUD/相机边界附近视觉拥挤。
- 底部留出约 80px，保留桌面边界感。

### Camera

保持当前 `Camera2D` limit 不变，让相机仍展示完整背景边界。

## 验收标准

1. 玩家不能走到背景最外边缘。
2. 玩家靠近左上/右下边界时不半露。
3. Dash 不会把玩家冲出安全区域。
4. `tests/smoke_test.gd` 输出 `AI_TEST_PASS`。
