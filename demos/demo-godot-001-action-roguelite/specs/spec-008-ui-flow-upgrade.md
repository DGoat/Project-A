# Spec: UI Flow Upgrade（开始/结算、赐福选择、HUD 侧边栏）

## 背景

当前 Demo 已完成 M1/M2，并进入 M3。战斗闭环可运行，祝福表现开始可感知，但 UI 流程仍偏测试态：

- 进入游戏后直接开打，没有开始界面。
- 胜利/失败只靠一行文本提示。
- 赐福选择只是简单按钮列表，缺少卡片感、构筑标签和选择反馈。
- HUD、Controls、Debug、已获得赐福分散，展示层级还不够清楚。

用户希望 UI 流程按以下顺序优化：

1. 开始 / 胜利 / 失败流程。
2. 赐福选择界面强化。
3. HUD 与侧边栏整理。

## 目标

- 增加可展示的开始界面。
- 增加更明确的胜利/失败结算界面。
- 强化赐福选择界面，让名称、效果、构筑方向更清楚。
- 整理 HUD 与侧边栏，让 Debug、已获得赐福、Controls 不干扰战斗。
- 保持当前战斗流程、敌人、祝福效果不变。

## 非目标

- 不做最终美术。
- 不做音效。
- 不做存档。
- 不做正式设置界面。
- 不做主菜单多页面。
- 不新增祝福。
- 不改敌人 AI 和房间配置。

## 阶段 1：开始 / 胜利 / 失败流程

### 设计

新增 `StartPanel`：

- 标题：`AI 协作动作 Roguelite Demo`
- 副标题：`清理房间，选择赐福，完成 3 房间挑战。`
- 按钮：`开始游戏`
- 简短操作提示：
  - 移动：WASD / 方向键
  - 攻击：鼠标左键 / J
  - 闪避：Space
  - 重开：R
  - Debug：键盘左上角 `` ` `` / `~` / `·` 所在键

新增 `ResultPanel`：

- 胜利标题：`胜利`
- 失败标题：`失败`
- 显示本局获得赐福列表。
- 按钮：`重新开始`

流程调整：

```text
进入场景 -> StartPanel
点击开始 -> 生成玩家和房间 1
死亡 -> ResultPanel(失败)
通关 -> ResultPanel(胜利)
点击重新开始 -> reload_current_scene
```

### 验收标准

- [ ] 进入游戏先看到开始界面，不直接生成战斗。
- [ ] 点击开始后进入房间 1。
- [ ] 死亡后显示失败结算面板。
- [ ] 通关后显示胜利结算面板。
- [ ] 结算面板能看到本局获得赐福。
- [ ] 重新开始按钮可用。
- [ ] `R` 重开仍可用。

## 阶段 2：赐福选择界面强化

### 设计

将当前三选一按钮强化为“卡片式文本按钮”：

每张卡片展示：

```text
[构筑标签]
赐福名称
效果描述
```

给现有赐福增加 `tags` 字段：

| 赐福 | 标签 |
|---|---|
| 锋刃打磨 | 直接伤害 |
| 迅捷出手 | 攻速 |
| 疾风步 | 闪避爆发 |
| 血暖余烬 | 生存 |
| 余火印记 | 燃烧 |
| 长柄之势 | 范围 |

选择反馈：

- 点击/按键选择后，Message 显示：`获得赐福：xxx`
- 已获得赐福面板同步追加。
- 进入下一房间前短暂保留提示。

### 验收标准

- [ ] 三选一界面显示构筑标签。
- [ ] 赐福名称和效果描述清楚。
- [ ] 选择后 Message 显示获得了哪个赐福。
- [ ] 已获得赐福面板同步更新。
- [ ] `1/2/3` 快捷选择仍可用。

## 阶段 3：HUD 与侧边栏整理

### 设计

HUD 分层：

左上角：

```text
HP | 房间 | 敌人
```

左上角下方：

```text
操作提示短文本
```

右上角 Debug/赐福侧边栏：

- 默认隐藏。
- 按键盘左上角 `` ` `` / `~` / `·` 所在键显示/隐藏。
- 显示时包含：
  - 已获得赐福列表
  - Debug 添加赐福按钮

清理原则：

- Debug 面板只服务开发测试。
- 默认不遮挡战斗。
- 非 Debug 状态下，画面只保留战斗必要 HUD。

### 验收标准

- [ ] 默认不显示右侧 Debug/赐福面板。
- [ ] 按键可显示/隐藏整个右侧面板。
- [ ] 显示后仍能点击 Debug 按钮添加赐福。
- [ ] 默认 HUD 不遮挡战斗区域。

## 风险

| 风险 | 影响 | 应对 |
|---|---|---|
| UI 状态变多导致流程 bug | 开始、死亡、胜利状态错乱 | 增加简单 `run_started` / `run_ended` 状态 |
| 结算和重开冲突 | 重复生成玩家/敌人 | 统一用 `reload_current_scene` 重开 |
| 赐福选择 UI 太占屏 | 遮挡战斗 | 只在清房后显示，战斗中隐藏 |
| Debug 面板和正式 UI 混淆 | 体验不清晰 | 默认隐藏，文档标明 Debug 用途 |

## 实施顺序

1. 实现开始 / 胜利 / 失败流程。
2. 实现赐福选择界面强化。
3. 整理 HUD 与侧边栏。
4. Godot 无头加载验证。
5. 用户复测。
6. 更新 Git。

## 回滚方案

如果 UI 流程引入严重问题：

1. 回滚 StartPanel / ResultPanel。
2. 保留当前可运行战斗闭环。
3. 赐福选择界面可单独保留或回滚。
4. Debug 面板保持隐藏逻辑不变。
