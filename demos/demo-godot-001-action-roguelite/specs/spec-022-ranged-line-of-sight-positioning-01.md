# Spec 022：远程敌人射线视野与攻击站位

## 背景

远程敌人当前只按距离寻路：距离远就靠近，距离近就停止或拉开。若远程敌人与玩家之间隔着硬障碍，投射物会被障碍挡住，但远程敌人仍可能站在原地射击，导致攻击无效。

## 目标

- 远程敌人只在与玩家之间有可射击直线时开火。
- 若直线被硬障碍挡住，远程敌人需要寻路到一个能看见/打到玩家的位置。
- 保留原有距离控制：太近拉开，太远靠近。

## 方案

1. 给 `RangedEnemy` 增加射线检测：
   - 使用 `PhysicsRayQueryParameters2D` 从远程敌人到玩家做 raycast。
   - 碰撞 mask 使用玩家层和地形层：`1 | 8`。
   - 若第一命中是玩家，则认为有射击视线。
   - 若第一命中是硬障碍，则认为没有射击视线。

2. 增加攻击站位查找：
   - 围绕玩家按多个角度采样点。
   - 优先选择距离接近 `preferred_distance`、可导航、且与玩家无遮挡的点。
   - 找不到时回退为导航到玩家位置。

3. 移动决策：
   - knockback 优先。
   - 无射击视线：寻路到攻击站位。
   - 有射击视线且太近：拉开。
   - 有射击视线且太远：靠近玩家。
   - 有射击视线且距离合适：停止并射击。

## 验证

- smoke test 增加远程敌人视线函数基础检查。
- 手测：在玩家与远程敌人之间放硬障碍时，远程敌人应绕到能打到玩家的位置再射击。
