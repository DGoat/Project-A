# Spec 012：Animation and UI Polish 0.1

## 背景

第一批玩具修理屋美术资源接入后，核心流程复测通过：移动、Dash、近战、远程、投射物、攻击、三选一、胜利、死亡重开均正常。当前主要短板转为表现层：角色和敌人缺少动画，UI 仍是 Godot 默认控件风格，和“童话治愈 × 玩具修理屋”主题不匹配。

本 Spec 目标是做一轮最小表现升级，不追求完整美术动画资产，而是用 Godot Tween、缩放、旋转、透明度、残影、面板样式等方式，让当前 Demo 从“静态贴图可玩”提升到“有基本动效和主题 UI”。

## 目标

- 补齐关键动作的伪动画反馈。
- 让 UI 明显摆脱默认控件感。
- 保持当前玩法、数值、房间流程不变。
- 不引入新的大规模美术资源。
- 不做构筑深化。
- 不做地图机关。

## 非目标

- 不制作帧动画 Sprite Sheet。
- 不接入骨骼动画。
- 不重做完整 UI 设计系统。
- 不调整敌人 AI 与战斗平衡。
- 不扩展房间数量。

## 改动范围

### 1. 主角动画/反馈

#### 移动伪动画

- 主角移动时加入轻微上下浮动或呼吸缩放。
- 停止移动时恢复默认 scale。
- 不旋转整个人，避免再次出现方向错误。

验收：

- 移动时角色不再像静态图片平移。
- 上下左右移动都不翻倒。

#### Dash 表现

- Dash 触发时：
  - 主角短暂变亮。
  - 生成 2-3 个淡化残影或使用短暂 scale 拉伸。
  - Dash Strike 充能时保留蓝色/高亮提示。

验收：

- 按 Space 时能明显看出“发条冲刺”。
- 不影响 Dash 距离和冷却。

#### 攻击表现

- 木尺攻击图出现时增加快速缩放/淡出。
- 保持当前已确认正确的上下反转方向。
- 攻击矩形提示可继续保留低透明度，辅助判定可读性。

验收：

- 攻击不再只是贴图闪现。
- 木尺方向保持当前正确版本。

#### 受击/死亡

- 受击：保留全屏红闪，同时主角短暂变红/抖动。
- 死亡：主角缩小或淡出，再触发失败界面。

验收：

- 受击能明确感知。
- 死亡不再是瞬间消失或只弹 UI。

### 2. 敌人动画/反馈

#### 近战敌人

- 移动时轻微 bob/缩放。
- 受击时短暂压缩/变亮。
- 死亡/修好时淡出或缩小。

#### 远程敌人

- 射击前摇：开火前 0.15-0.25 秒变亮或轻微放大。
- 射击时产生小后坐力缩放。

#### 投射物

- 保留旋转。
- 可增加轻微发光/颜色脉冲。

验收：

- 近战/远程敌人的行动状态更容易区分。
- 远程射击有预兆。

### 3. UI 视觉升级

#### 全局风格方向

关键词：

```text
夜晚修理屋、暖灯、木质面板、布片边框、纽扣、纸牌、柔和阴影、童话绘本
```

基础颜色：

```text
深棕：#2A1B14
木桌棕：#6A3F24
暖灯黄：#F5C76B
布片蓝：#6C8FA6
布片红：#A75D4C
奶油白：#F6E8C8
透明黑：rgba(20, 12, 8, 0.68)
```

#### HUD

- 状态文本加半透明底或暗色描边。
- 操作提示降低存在感，避免压画面。
- 房间信息与 HP 分成更清晰层级。

#### 开始界面

- 面板改为暖色半透明木质/布片风格。
- 标题更大，按钮更像“木牌/布牌”。

#### 修理灵感三选一

- 三个按钮改为卡片风格。
- 每张卡片突出：标签、名称、描述。
- Hover/按下状态更明显。

#### 结算界面

- 胜利/失败标题更主题化。
- 本晚灵感列表更易读。

#### Debug 面板

- 保持可用，但视觉上区别于正式 UI。
- 默认仍隐藏，通过 ` / ~ / · 所在键切换。

验收：

- UI 不再明显是 Godot 默认控件。
- 开始、三选一、结算三个关键界面更符合玩具修理屋主题。
- 不影响点击、按键选择和 Debug 面板。

## 实现建议

### 动画实现

优先使用：

- `Tween`
- `modulate`
- `scale`
- `position`
- 少量临时 Sprite 残影

避免：

- 新增复杂 AnimationPlayer 状态机。
- 大量硬编码一次性节点。
- 需要额外图像资源的方案。

### UI 实现

优先使用：

- `Theme` 或直接给关键 Panel/Button 加 `StyleBoxFlat`。
- 统一颜色常量写在 `main.gd` 或后续拆到主题资源。
- 先改关键面板，不做全项目 UI 框架。

## 验收清单

1. 玩家移动有轻微动态反馈。
2. Dash 有明显发条冲刺感。
3. 攻击木尺有快速动效，方向仍正确。
4. 玩家受击比当前更明显。
5. 敌人受击/死亡有反馈。
6. 远程敌人射击前有提示。
7. 开始界面不再像默认 UI。
8. 三选一修理灵感卡片可读性提升。
9. 胜利/失败界面更符合主题。
10. Debug 面板仍可隐藏/显示。
11. 三房间流程、胜利、死亡重开不被破坏。
12. `tests/smoke_test.gd` 仍输出 `AI_TEST_PASS`。

## 风险

- 伪动画过多可能影响攻击/受击可读性，需要保持克制。
- UI 样式若只靠代码生成，后续可能难维护；本轮先接受，后续可拆 Theme 资源。
- Dash 残影若不回收会产生节点泄漏，需确保自动 fade 后 `queue_free()`。
