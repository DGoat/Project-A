# Spec: Toy Visual Pass 0.1（玩具修理屋基础视觉占位）

## 背景

当前 Demo 已完成玩具修理屋主题替换层：

- 赐福改为修理灵感。
- 攻击改为修理打击。
- Dash 改为发条冲刺。
- 敌人改为失控玩具。
- 胜利/失败文案改为童话修理主题。

但画面仍主要是抽象几何：

- 玩家是绿色箭头。
- 近战敌人是红色形状。
- 远程敌人是蓝色形状。
- 场景是冷黑色竞技场。
- 投射物和攻击提示还没有玩具/修理语义。

因此下一步不是深化机制，而是先做基础视觉占位，让玩家“看得出”这是玩具修理屋方向。

## 目标

- 不引入外部美术资源。
- 只使用 Godot 基础 2D 节点和几何形状。
- 将当前抽象图形替换为更有主题语义的占位表现。
- 保持所有玩法逻辑不变。
- 保持当前性能和脚本结构简单。

## 非目标

- 不做最终美术。
- 不做动画系统。
- 不做粒子系统。
- 不做音效。
- 不新增机制。
- 不新增敌人类型。
- 不修改数值平衡。

## 视觉方向

关键词：

```text
夜晚修理屋 / 暖灯 / 木头 / 布偶 / 发条 / 积木 / 纽扣 / 小工具
```

整体色彩：

| 元素 | 当前 | 新方向 |
|---|---|---|
| 背景 | 冷黑 | 暖暗棕 / 夜间修理台 |
| 玩家 | 高饱和绿 | 暖黄小灯 + 青绿色围裙 |
| 近战敌人 | 红色 | 木块红 / 积木兵 |
| 远程敌人 | 蓝色 | 铁皮蓝 / 弹簧炮 |
| 投射物 | 普通弹体 | 纽扣 / 螺丝 |
| 攻击提示 | 黄色矩形 | 木尺 / 工具挥击 |

## 具体改动

### 1. 场景：夜晚修理台

修改 `Main.tscn` 中 `Arena` 颜色：

```text
从冷黑色 -> 暖暗棕色
```

目标：让场景更像夜晚修理台/玩具箱，而不是普通黑色战斗场。

可选增加：

- 不新增节点。
- 仅改颜色，保持轻量。

### 2. 玩家：小修理师占位

当前玩家 Body 是箭头三角形。

短期方案：

- 保留朝向箭头功能。
- 调整颜色为“暖灯 + 围裙”风格。
- 增加一个小工具感：让箭头更像拿着工具的朝向形状。

建议：

| 节点 | 改动 |
|---|---|
| `Player/Body` | 颜色改为青绿色围裙感 |
| `AttackPreview` | 改成木尺/工具扫过的暖木色 |
| `HealthBarFill` | 保持绿色或改为温暖黄绿 |

### 3. 近战敌人：积木兵 / 断线木偶

当前近战敌人红色。

短期方案：

- 改为木块红 / 暗红木色。
- 形状保持简单但更像积木块。
- 精英近战用更深/更大的颜色。

可接受占位：

```text
红棕色三角/方块 = 失控积木兵
```

### 4. 远程敌人：弹簧炮玩具

当前远程敌人蓝色。

短期方案：

- 改为铁皮蓝。
- 保持朝向旋转。
- 视觉上代表远程发射玩具。

### 5. 投射物：纽扣 / 螺丝

当前 Projectile 是简单形状。

短期方案：

- 改颜色为金属灰 / 纽扣黄。
- 如果当前是圆形，保留圆形并称作“纽扣弹”。
- 不改碰撞逻辑。

### 6. 攻击提示：木尺 / 工具挥击

当前攻击提示为半透明黄色矩形。

短期方案：

- 改为暖木色半透明矩形。
- 继续和实际判定同步。
- 语义：小木锤/木尺扫过区域。

## 验收标准

- [ ] 场景颜色更像夜晚修理屋/玩具箱。
- [ ] 玩家不再像纯绿色三角，而更像小修理师占位。
- [ ] 近战敌人更像失控积木兵/木偶占位。
- [ ] 远程敌人更像弹簧炮/铁皮玩具占位。
- [ ] 投射物更像纽扣/螺丝。
- [ ] 攻击提示更像木尺/工具挥击。
- [ ] 受击、燃烧、Dash Strike、回血闪色仍可见。
- [ ] 3 房间流程仍可通关。
- [ ] Godot 运行无脚本错误。

## 风险

| 风险 | 影响 | 应对 |
|---|---|---|
| 颜色太暖导致可读性下降 | 敌我不清 | 保持玩家、近战、远程色相区分 |
| 主题色和反馈色冲突 | 受击/燃烧不明显 | 保留红闪、橙闪、青色 Dash Strike |
| 只改颜色仍不够像玩具 | 主题感有限 | 后续再增加简单符号，如纽扣孔、发条小尾巴 |

## 实施步骤

1. 修改 `Main.tscn` Arena 色彩。
2. 修改 `Player.tscn` 玩家与攻击提示色彩。
3. 修改敌人/投射物场景色彩。
4. 更新脚本中的颜色恢复值。
5. Godot 无头加载验证。
6. 用户复测视觉方向。
7. 同步 Git。

## 回滚方案

如果视觉可读性下降：

1. 保留主题文案。
2. 回滚场景和角色颜色。
3. 单独重新设计颜色方案。
