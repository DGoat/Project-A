# Spec: M1 可玩性修复 - 角色可视距离与攻击范围提示

## 背景

M1 手测中暴露两个影响基础可玩性的问题：

1. 血量 UI 与角色距离太远，玩家需要频繁看左上角，战斗中难以感知自身血量。
2. 看不到自己的攻击范围，只能通过敌人是否掉血判断是否命中。

这两个问题直接影响 M1 验证：玩家无法清楚判断自身状态和攻击是否有效。

## 目标

- 让玩家在角色附近看到自身血量。
- 让玩家在攻击时看到攻击范围。
- 不引入复杂 UI 系统。
- 不做完整美术表现，只做可玩性可视化。

## 非目标

- 不做最终 HUD 美术。
- 不做完整伤害数字系统。
- 不做完整受击/命中特效系统。
- 不做复杂动画。
- 不重构战斗系统。

## 范围

本次只处理两个可视化问题：

1. 玩家头顶血条或血量文本。
2. 攻击范围提示。

如果顺手发现轻微参数问题，可以记录，但不在本 Spec 中扩展处理。

## 设计方案

### 1. 角色附近血量显示

在 `Player.tscn` 下增加一个简单血条或文本节点。

优先方案：

```text
Player
├─ Body
├─ CollisionShape2D
├─ AttackArea
└─ HealthBarRoot
   ├─ HealthBarBg
   └─ HealthBarFill
```

表现规则：

- 血条显示在玩家上方。
- 血条跟随玩家移动。
- 受伤或回血时更新宽度。
- 仍保留左上角 HUD，用于调试。

M1 阶段先用矩形色块：

- 背景：暗红/黑色
- 当前血量：绿色或红色

### 2. 攻击范围提示

在 `Player.tscn` 下增加一个攻击范围显示节点。

优先方案：

```text
AttackPreview: Polygon2D 或 ColorRect/Line2D 简易形状
```

表现规则：

- 攻击时短暂显示。
- 显示位置与 `AttackArea.position = facing * attack_range` 保持一致。
- 显示时间与攻击判定窗口一致，约 `0.08s`。
- 颜色使用半透明黄色/白色。

M1 阶段不追求扇形攻击，只显示一个半透明圆形或方块，先让玩家知道“这一击打哪里”。

## 数据结构 / 配置

暂不新增数据文件。

建议在 `player.gd` 中保留简单变量：

```gdscript
var attack_preview_time := 0.08
var health_bar_width := 36.0
```

## 验收标准

- [ ] 玩家角色上方能看到血条。
- [ ] 血量变化时，角色上方血条同步变化。
- [ ] 攻击时能看到攻击范围提示。
- [ ] 攻击范围提示方向跟随玩家朝向。
- [ ] 不影响原有攻击命中判定。
- [ ] 不影响房间流程。
- [ ] 运行无脚本报错。

## 风险

| 风险 | 影响 | 应对 |
|---|---|---|
| 攻击提示和真实判定范围不一致 | 玩家误判攻击 | 提示位置直接复用 `attack_area.position` 和 `attack_range` |
| 血条跟随玩家旋转 | 显示怪异 | 血条不要作为 `Body` 子节点，放在 Player 下独立节点 |
| UI 过度遮挡角色 | 影响战斗观察 | 血条保持小尺寸，放在角色上方 |
| Godot 场景手写节点格式出错 | 项目打不开 | 小步修改，修改后用 Godot 验证 |

## 实施步骤

1. 更新 `issue-tracker.md`，新增两个问题：
   - `UI-001`：血量 UI 离角色太远。
   - `FEEL-005`：看不到攻击范围。
2. 修改 `Player.tscn`，增加血条和攻击范围提示节点。
3. 修改 `player.gd`：
   - 初始化血条。
   - `take_damage()` 和 `heal()` 后更新血条。
   - `attack()` 时显示攻击范围提示，短暂后隐藏。
4. 在 Godot 中运行验证。
5. 更新 `dev-log.md`。
6. 必要时更新 `pitfalls.md`。

2. 回退 `player.gd`。
3. 保留 issue 记录，改为后续处理。
