# Spec 023：差异化核心体验与房间概率方向

## 背景

当前 Demo 已验证基础动作 Roguelite 闭环，但仍需要明确与参考作品的核心体验差异。

参考维度：

- 空洞骑士：战斗 + 跳跳乐，长线探索型玩法。
- 暖雪 / 哈迪斯：固定 Boss，短流程，每局不同强化不同体验。
- 杀戮尖塔：随机 Boss，短流程，每局不同强化不同体验。
- 以撒：固定 Boss，长流程，带探索与弹幕躲避，每局不同强化不同体验。

## 差异化方向

本项目可将核心差异化设定为：

- 随机 Boss，对应随机地图主题。
- 短流程。
- 每局不同强化带来不同体验。
- 强化路线不只改变战斗数值，也改变后续房间出现概率。

## 核心机制假设

玩家获得不同类型强化后，对应类型房间的出现概率提升。

设计意图：

- 构筑路线会改变玩家后续遇到的问题。
- 对应路线能力，往往是解决对应房间问题的最优解。
- 玩家不只是“拿强度”，也是在选择接下来更可能面对的修理难题。

## 初始路线映射

| 构筑路线 | 房间问题倾向 | 能力最优解倾向 |
|---|---|---|
| 发条 | 高机动敌人、节奏机关、弹簧垫 | 攻速、连击、冲刺爆发 |
| 缝补 | 高伤害压力、持续消耗、救援型目标 | 回血、护盾、容错 |
| 胶水 | 大量敌人、追击压力、区域封锁 | 减速、定身、区域控制 |
| 积木 | 障碍密集、可破坏物、地形互动 | 范围、击退、障碍互动 |
| 灯光 | 暗区、诅咒、持续污染 | 净化、安全区、持续恢复 |

## 后续任务

- 设计房间类型权重公式。
- 设计随机 Boss 与地图主题映射。
- 设计每条构筑路线至少 3 个可感知强化。
- 设计每类房间至少 2 个“能力最优解”场景。
