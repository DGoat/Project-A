# Spec: M3 调试赐福面板、已获得赐福面板与受伤全屏红闪

## 背景

M3 复测中，`Long Reach` 和 `Wind Step` 已确认可见，Demo 仍可通关。

当前测试痛点：

- 想验证指定赐福时，需要依赖清房后的随机三选一。
- `余火印记`、`血暖余烬` 等效果不一定能及时抽到。
- 当前 Run 已获得哪些赐福缺少记录面板。
- 玩家受伤反馈目前只有角色闪色，战斗中不够强。

## 目标

- 增加调试赐福面板，允许点击按钮给玩家添加指定赐福。
- 增加已获得赐福面板，实时记录当前 Run 拿到的赐福。
- 赐福名称和描述改为中文。
- 玩家受伤时增加全屏红色闪烁，类似 MOBA/动作游戏受伤反馈。
- 保持当前 3 房间流程不变。

## 非目标

- 不做正式 Debug Console。
- 不做祝福选择 UI 重构。
- 不做保存/清除赐福功能。
- 不做完整屏幕特效系统。
- 不做音效。
- 不保留 `4-9` 测试热键。

## 设计方案

### 1. 调试赐福面板

右侧增加 Debug 面板按钮，点击添加现有 6 个赐福：

| 按钮 | 赐福 |
|---|---|
| 锋刃打磨 | 攻击伤害 +20% |
| 迅捷出手 | 攻击冷却 -15% |
| 疾风步 | 闪避后，下一次攻击伤害 +50% |
| 血暖余烬 | 击杀敌人后回复 8 点生命 |
| 余火印记 | 攻击会附加持续燃烧伤害 |
| 长柄之势 | 攻击范围 +20% |

### 2. 已获得赐福面板

在右上角增加 `AcquiredBlessingsPanel`：

- 标题：`赐福`
- 初始显示：`暂无`
- 每次通过三选一或 Debug 面板获得赐福后，追加赐福名。
- 重开后清空。

### 3. 受伤全屏红闪

在 `Main.tscn` 的 UI 下新增 `DamageFlash`：

- 类型：`ColorRect`
- 全屏锚点
- 初始透明
- 颜色：半透明红色
- 鼠标过滤：忽略

玩家受伤时：

1. `player.gd` 发出 `damaged` signal。
2. `main.gd` 接收后显示红色 overlay。
3. 使用 Tween 快速淡出。

## 涉及文件

| 文件 | 改动 |
|---|---|
| `data/blessings.json` | 赐福名称和描述改中文 |
| `scripts/player.gd` | 增加 `damaged` signal，受伤时发出 |
| `scripts/main.gd` | 监听受伤信号，处理 Debug 面板、已获得赐福列表和红闪 |
| `scenes/Main.tscn` | 增加全屏 `DamageFlash`、`AcquiredBlessingsPanel`、`DebugPanel` |
| `project.godot` | 移除测试热键 InputMap |
| `controls.md` | 记录 Debug 面板 |
| `dev-log.md` | 记录实现过程 |
| `issue-tracker.md` | 记录测试便利性与受伤反馈问题 |

## 验收标准

- [ ] 点击 Debug 面板按钮能手动添加对应赐福。
- [ ] 添加赐福后 UI Message 显示当前添加的赐福名。
- [ ] 右上角面板能显示已获得赐福。
- [ ] 通过三选一获得赐福后，面板追加记录。
- [ ] 通过 Debug 面板获得赐福后，面板追加记录。
- [ ] 三选一赐福名称与描述为中文。
- [ ] 玩家受伤时有全屏红色闪烁。
- [ ] 红闪会快速淡出，不持续遮挡。
- [ ] 3 房间流程仍可通关。
- [ ] Godot 运行无脚本错误。

## 风险

| 风险 | 影响 | 应对 |
|---|---|---|
| Debug 面板被误认为正式玩法 | 设计混乱 | 文档明确标记为调试用途 |
| 红闪遮挡太强 | 影响战斗判断 | 初始透明度控制在 0.28 左右，快速淡出 |
| Tween 叠加 | 多次受伤闪烁异常 | 每次触发前 kill 旧 Tween |

## 回滚方案

如果影响体验：

1. 保留已获得赐福面板，关闭 Debug 面板。
2. 或只保留红闪，移除 Debug 面板。
3. 最终可整体回滚本 Spec 涉及改动。
